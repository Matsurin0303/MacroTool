# CSV互換ポリシー（Macro_v1.0.0 本版）

## 1. 目的

本ドキュメントは、**Macro_v1.0.0 本版で実装対象とする CSV 互換範囲**を定義する。

本版では、将来のCSVメジャー差分自動変換までは扱わず、**現在版で確実に実装・検証する範囲だけ**を正式仕様とする。

## 2. 本版の正式スコープ

### 2.1 対応対象

- Import 対応: **`CSV_v1.0` のみ**
- Export 対応: **`CSV_v1.0` のみ**
- スキーマ識別方法: **ヘッダ完全一致**

### 2.2 本版で対象外とする内容

以下は **将来予定** とし、Macro_v1.0.0 の実装対象外とする。

- `CSV_v1.x` から `CSV_v2.x` への自動マイグレーション
- メタ行（例: `#SchemaVersion=...`）を用いたスキーマ識別
- 複数段階のスキーマ変換（例: `v1 -> v2 -> v3`）
- 旧スキーマ / 新スキーマの相互変換
- 旧CSVスキーマへのダウングレードExport

## 3. Import 互換ポリシー

### 3.1 受け入れ条件

Import は、以下をすべて満たす CSV のみ受け付ける。

- 先頭行がヘッダ行である
- ヘッダが `CSV_v1.0` の固定ヘッダと **完全一致**する
- 各列値が `CSV_v1.0` の制約を満たす

### 3.2 Import エラー条件

以下に該当する場合は Import エラーとする。

- ヘッダが `CSV_v1.0` と一致しない
- 先頭にメタ行が存在する
- 未知列、欠落列、順序違いがある
- `CSV_v1.0` の型・enum・参照制約を満たさない

### 3.3 自動変換の扱い

本版では、**Import 時の自動変換は行わない**。

- `CSV_v1.0` 以外は受け入れない
- 将来バージョン形式を読めたとしても、本版仕様ではサポートしない
- 変換ロジックを仮実装・暫定実装として入れない

## 4. Export ポリシー

- Export は常に **`CSV_v1.0` 固定ヘッダ** で出力する
- メタ行は出力しない
- 将来スキーマ向けの追加列・補助行は出力しない

## 5. バージョン変更時の扱い

将来 `CSV_v2.0` 以降を導入する場合は、以下を **別チケット / 別仕様** として追加する。

- `Version_Mapping_Table.md` の正式更新
- `Migrations/CSV_vN_to_vN+1.md` の追加
- Import 側のスキーマ判定方式見直し
- 自動変換のエラー / 警告ポリシー追加

本ドキュメント自体は、**Macro_v1.0.0 本版の互換範囲定義**として扱う。

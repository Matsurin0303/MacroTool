# TKT-009 修正サマリ

## 対応概要
`02_Requirements/Functional_Spec.md` において、`2-2-1-1*` および `3-2-1-1*` の `New Record` 実行時未保存確認ダイアログを v7 基準へ修正した。

## 修正内容
- `2-2-1-1` の起動条件を `Recent Files` ではなく `New Record` 実行時へ修正
- `2-2-1-1-1` / `2-2-1-1-2` の遷移先を、`Recent Files` 読込ではなく `新規記録開始` へ修正
- `2-2-1-1-3` に `New Record` を中止して現在画面へ留まる旨を明記
- `3-2-1-1-3` に `New Record` を中止して現在画面へ留まる旨を明記
- 共通ルールとして、`New Record` 実行時の未保存確認も `保存` / `保存しない` / `キャンセル` の分岐ルールに従うことを追記

## 完了条件に対する確認
- `2-2-1-1*` と `3-2-1-1*` がどちらも `New Record` 実行前の未保存確認として記載されている
- `保存` / `保存しない` / `キャンセル` ごとの遷移が明示されている
- `Recent Files` 読込の誤記は除去されている

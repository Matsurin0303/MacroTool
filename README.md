以下は本プロジェクトのバージョニングおよびリリース運用規約です。

## Versioning / リリース運用

本プロジェクトは `vX.Y.Z` でバージョン管理します。

- **X**：互換性が壊れる変更（Breaking Change）
  - 例：既存マクロファイルが読み込めない / 同じマクロが異なる挙動になる / 設定の意味が変わる 等
- **Y**：互換性を壊さない機能追加・拡張（Feature）
  - 例：新アクション追加、UI項目追加、オプション追加 等（既存マクロは従来通り動作）
- **Z**：バグ修正・軽微な改善（Fix）
  - 例：例外で落ちる、誤動作、文言や表示崩れ、安定性改善 等

### Git Tag / Release
- リリースタグは `Macro_vX.Y.Z` を使用します（例：`Macro_v2.1.3`）
- タグは必ず `master` のリリース確定コミットに付けます
- GitHub Release は `Macro_vX.Y.Z` と一致させます

### Docs（仕様書）の扱い
- リリースタグを切る時点で `/docs` は最新で凍結（Freeze）されていること
- 仕様書は `/docs/00_Index.md` を入口とし、参照先を固定します
- 将来実装予定の機能は本版では対象外とし、仕様書には **「対象外（将来予定）」** と明記します

### Branch policy（推奨）
- `master`：常に安定
- `feature/*`：互換性維持の機能追加（主に Y を上げる変更）
- `bugfix/*`：バグ修正（主に Z を上げる変更）
- `breaking/*`：互換性破壊を含む変更（主に X を上げる変更）
